# دليل شرح مشروع Brand Name Suggester 🚀

أهلاً بيك! الدليل ده معمول عشان يشرحلك مشروع **Brand Name Suggester** (مولد أسماء العلامات التجارية العربية) بأسلوب مصري بسيط جداً وواضح، من غير أي تعقيدات تقنية.

---

## 1. مكتبات المشروع (package.json) 📦

### المشروع ده شغال بإيه؟
المشروع شغال بتقنيات حديثة جداً وسريعة:
* الـ **React 19:** مكتبة الواجهات الشهيرة لبناء الصفحات والمكونات التفاعلية.
* الـ **Vite 6:** أداة بناء وتطوير فائقة السرعة بتشغل المشروع وتحدث التغييرات في لحظتها.
* الـ **Express & Node.js:** سيرفر خلفي (Backend) بياخد الطلبات ويتواصل مع الذكاء الاصطناعي.
* الـ **TypeScript:** لغة بتضيف أنواع وبيانات صارمة فوق JavaScript عشان تمنع الأخطاء أثناء الكتابة.
* الـ **Tailwind CSS v4:** لتنسيق الألوان والأشكال بشكل شيك وعصري.

### أهم المكتبات المستخدمة ودور كل واحدة:
1. الـ **`@xyflow/react` (React Flow):** أهم مكتبة في الواجهة، وهي المسؤولة عن رسم الشجرة التفاعلية والـ Nodes (العقد) والـ Edges (الروابط) على شاشة Canvas تقدر تحركها وتكبر وتصغر فيها.
2. الـ **`@google/genai`:** المكتبة الرسمية للتواصل مع نماذج ذكاء Google Gemini (زي Gemini Flash) لتوليد الأسماء والأفكار.
3. الـ **`openai`:** مكتبة للتواصل مع أي خدمات أو مزودين ذكاء اصطناعي بيستخدموا صيغة OpenAI (زي Groq أو DeepSeek أو Ollama).
4. الـ **`motion` (`motion/react`):** مكتبة أنيميشن بتعمل حركات ناعمة والتنقلات السلسة بين المكونات والقوائم.
5. الـ **`lucide-react`:** مجموعة أيقونات عصرية وخفيفة بتستخدم في كل الأزرار والقوائم.
6. الـ **`clsx`:** مكتبة صغيرة بتساعدنا ندمج فئات التنسيق (CSS Classes) بشرط مرن وسهل.
7. الـ **`dotenv`:** عشان تقرأ المفاتيح السرية والمتغيرات من ملف `.env`.

## 3. نقطة الدخول (Entry Point) وتدفق التشغيل 🏁

إزاي التطبيق بيبدأ يشتغل من أول ما نكتب أمر التشغيل لحد ما الصفحة تفتح؟

1. **تشغيل السيرفر (`server.ts`):**
   لما بتكتب `pnpm dev` أو `npm run dev` في الترمنال، بيتم تشغيل ملف `server.ts` عن طريق أداة `tsx`.
2. **إنشاء الـ API (`services/api-app.ts`):**
   ملف `server.ts` بيشغل سيرفر Express وبيستدعي دالة `createApiApp()` اللي بتجهز الـ Endpoints زي `/api/suggest` و `/api/transliterate`.
3. **ربط Vite Middleware:**
   في وضع التطوير (Development)، سيرفر Express بيمسك Vite كـ Middleware عشان يخدم ملفات الصفحات والـ HMR (التحديث اللحظي للفيجوال).
4. **تحميل صفحة الـ HTML (`index.html`):**
   لما تفتح المتصفح على `http://localhost:3000` السيرفر بيرجع ملف `index.html`.
5. **تشغيل السكربت الرئيسي (`src/main.tsx`):**
   ملف `index.html` فيه سطر بيطلب `/src/main.tsx`.
6. **تركيب React في الـ DOM:**
   ملف `src/main.tsx` بيستخدم `createRoot` عشان يركّب المكون الرئيسي `<App />` من ملف `src/App.tsx` داخل عنصر الـ `<div id="root"></div>`.
7. **عرض الواجهة للمستخدم:**
   المكون `<App />` بيشوف لو مفيش كلمة بداية يفتح صفحة الهبوط `<LandingPage />` عشان المستخدم يكتب أول كلمة (Seed Word)، وبعد ما يكتبها ينتقل تلقائياً لكانفاس الشجرة التفاعلية `<ExplorationTree />`.

## 4. نظام التنقل والصفحات (Routing) 🗺️

* **هل المشروع بيستخدم مكتبة توجيه زي `react-router`؟**
  لا! المشروع مصمم كصفحة واحدة تفاعلية بالكامل (Single Page Application - SPA) بدون أي مكتبات routing خارجية.

* **إزاي التنقل بيحصل بين الصفحات؟**
  التنقل بيحصل عن طريق **العرض الشرطي (Conditional Rendering)** جوه ملف `src/App.tsx` بناءً على قيمة حالة (State) اسمها `rootWord`:
  * **الصفحة الأولى (Landing Page):** لو قيمة `rootWord == null` (يعني لسة مفيش كلمة جذر)، التطبيق بيعرض مكون `<LandingPage />`.
  * **صفحة الشجرة الاستكشافية (Exploration Tree):** لما المستخدم يكتب كلمة بداية ويضغط على زِر الاستكشاف، قيمة `rootWord` بتبقى اسم الكلمة، وفوراً التطبيق بيخفي صفحة الهبوط ويعرض شجرة الاستكشاف `<ExplorationTree />`.

## 5. إدارة البيانات والحالة العامة (State Management) 🔄

* **هل فيه مكتبات إستيت عامة زي Redux أو Zustand أو Context؟**
  لا يوجد Redux أو Zustand.

* **إزاي البيانات بتتنقل بين الصفحات والمكونات؟**
  1. **الـ React Hooks المباشرة:** البيانات بتدار داخل المكون الرئيسي `src/App.tsx` باستخدام `useState` و `useEffect` و `useRef`.
  2. **الـ Props و الـ Callbacks:** لما نحتاج نبعث بيانات أو وظائف للمكونات الفرعية، بنبعتها عن طريق الـ Props (زي ما `App.tsx` بيبعت الكلمة الجذرية `rootWord` ودالة حفظ المفضلة `onSelectWord` لـ `ExplorationTree`).
  3. **الذاكرة المحلية للمتصفح (`localStorage`):** بنستخدمها لحفظ الإعدادات والمفضلة وجلسة العمل:
     * حفظ مفتاح وإعدادات الذكاء الاصطناعي في `brand_suggester_ai_provider`.
     * حفظ آخر جلسة شجرة استكشاف في `brand_tree_last_session` عشان لو قفلت الصفحة ورجعت تلاقي شغلك موجود زي ما هو.

## 6. شرح أهم 3 ملفات في المشروع بالتفصيل 🧠

اخترنا أهم 3 ملفات بيشكلوا العصب الحقيقي للمشروع:

### 1. [services/new-brand-suggester.ts](file:///d:/programing/JavaScript/React/brand-name-suggester/services/new-brand-suggester.ts) (محرك الذكاء الاصطناعي)
ده القلب المدبر للمشروع والمسؤول عن توليد الأسماء والتحويلات اللغوية:
* **التحقق من الكلمة (`isValidArabicWord`):** دالة بتتأكد إن المدخل كود عربي ومش فارغ ولا محاولة اختراق (Prompt Injection).
* **إرسال الطلبات للمزود المخصص (`generateWithProvider`):** دالة بتاخد الأوامر (Prompts) وتبعثها لأي سيرفر ذكاء اصطناعي داعم لـ OpenAI API وترجع النتيجة.
* **استخراج الـ JSON الصريح (`extractJson`):** دالة بتنظف النتيجة وترجع كود JSON نظيف حتى لو النموذج حط Markdown code fences.
* **توليد الأسماء (`suggest_brand_names`):** الدالة الرئيسية اللي بتاخد الكلمة وعدد الحروف والنبرة (Tone) والمود (اشتقاقات أو جمع)، وبتصيغ Prompt دقيق للذكاء الاصطناعي وتستلم قائمة بالأسماء المقترحة مع ترجمتها اللاتينية (Transliteration).
* **الدوال المتخصصة:**
  * الـ `transliterate_word`: لتحويل الكلمة العربية إلى نطق لاتيني بحروف كابيتال (زي: "روضة" -> "RAWDAH").
  * الـ `extractSynonyms`: لجلب المرادفات العربية للكلمة.
  * الـ `extractAntonyms`: لجلب عكس الكلمة أو الأضداد.
  * الـ `extractNisba`: لتوليد أسماء النسبة (زي: "بحر" -> "بحري").
  * الـ `extractRhymes`: لجلب كلمات على نفس القافية والوزن.
  * الـ `suggestCompoundNames`: لاقتراح أسماء مركبة من كلمتين.

### 2. [src/components/ExplorationTree.tsx](file:///d:/programing/JavaScript/React/brand-name-suggester/src/components/ExplorationTree.tsx) (كانفاس الشجرة التفاعلية)
ده الكومبوننت المسؤول عن رسم الشجرة الشجرية واستعراض العقد تفاعلياً:
* **مكتبة React Flow:** بيستورد المكونات الرئيسية `@xyflow/react` لرسم المساحة التفاعلية (Canvas) ويوصل بين العقد بخطوط.
* **إدارة العقد والروابط (`useNodesState` & `useEdgesState`):** بيخزن قائمة جميع العقد (Nodes) والروابط (Edges) الموجودة على الشاشة.
* **خوارزمية منع التداخل (`resolveOverlaps`):** خوارزمية ذكية بتحسب مكان تفريع العقد الجديدة بشكل دائري (Radial Spacing)، وتأكد إن الأبناء الجداد ميتداخلوش فوق الكروت القديمة.
* **توسيع الكلمة (`handleExpandNode`):** لما تضغط على زرار التوسع في أي عقدة، الدالة دي بتعمل طلب لسيرفر الـ Backend `/api/suggest` يجيب الكلمات الجديدة، وتضيف عقد جديدة (Child Nodes) مربوطة بالعقدة الأم بخطوط ناعمة.
* **شريط الأدوات الأسطوري:** بيوفر أزرار للتكبير/التصغير، مسح الشجرة، إعادة التوليد، حفظ الشجرة كصورة عالية الدقة، وتلقائياً بيحفظ حالة الشجرة في `localStorage`.

### 3. [src/components/BrandNode.tsx](file:///d:/programing/JavaScript/React/brand-name-suggester/src/components/BrandNode.tsx) (كارت الكلمة التفاعلي)
ده الكومبوننت المسؤول عن شكل وشعور كل كلمة بتظهر جوه الشجرة:
* **الأزرار القمَرية (Satellite Controls):** أزرار دائرية صغيرة بتظهر حوالين الكارت لما تقف عليه بالماوس، بتسمحلك تختار عدد الحروف المفضل، والنبرة (Tone) المطلوب التوليد بيها قبل ما تضغط تفريع.
* **عرض الكلمة والترجمة اللاتينية:** بيعرض الكلمة العربية بخط واضح مع تشكيلها، والترجمة بالإنجليزية تحتها مباشرة.
* **التعديل المباشر (Inline Edit):** بيسمح للمستخدم يضغط على الكلمة ويغيرها يدوياً لو حابب.
* **القائمة الإضافية "..." (More Menu):** قائمة تفاعلية بتسمح باكتشاف:
  * **الأصداء اللغوية (Cross-lingual Echoes):** معرفة الكلمة دي معناها إيه بلغات تانية (زي الفارسية أو الأوردو أو غيرها).
  * **المرادفات والأضداد والقوافي:** توليد تفريعات بناءً على المعنى أو الوزن القافي.
